Explication de modèles NLP¶

Tous les modles d'apprentissage automatique qui oprent dans des dimensions suprieures ce qui peut tre directement visualis par l'esprit humain peuvent tre qualifis de modles bote noire, ce qui affecte l'interprtabilit des modles. En particulier dans le domaine du traitement du langage naturel (NLP), il est toujours le cas que les dimensions des caractristiques sont trs grandes, ce qui rend beaucoup plus complique l'explication de l'importance des caractristiques.

Ainsi ce notebook concistera construire un modle de classification de texte multi-classes, puis appliquer sparment LIME et SHAP pour l'explication et l'interprtabilit des modles.

from bs4 import BeautifulSoup
from nltk.corpus import stopwords
import re

REPLACE_BY_SPACE_RE = re.compile('[/(){}\[\]\|@,;]')
BAD_SYMBOLS_RE = re.compile('[^0-9a-z #+_]')
STOPWORDS = set(stopwords.words('english'))

def clean_text(text):
    text = BeautifulSoup(text, "lxml").text # Dcode le HTML en texte brut
    text = text.lower() # Convertit en minuscules
    text = REPLACE_BY_SPACE_RE.sub(' ', text) # Remplace certains symboles par des espaces
    text = BAD_SYMBOLS_RE.sub('', text) # Supprime certains symboles indsirables
    text = ' '.join(word for word in text.split() if word not in STOPWORDS) # Supprime les stopwords 
    return text

Dans cette tape, nous allons prparer les tiquettes pour l'entranement en utilisant la couche StringLookup de TensorFlow, qui convertit les labels sous forme de texte en encodages numriques (multi-hot) que le modle peut utiliser.

Multi-hot : Chaque label est reprsent par un vecteur binaire o chaque dimension correspond un label possible. Si un document est associ plusieurs labels, ces labels sont tous activs dans le vecteur multi-hot. Cela permet au modle de traiter des problmes de classification multi-label, o un chantillon peut appartenir plusieurs catgories simultanment.

On extrait le vocabulaire (les labels uniques) de la couche StringLookup. Ensuite, on nettoie les entres du vocabulaire en supprimant les espaces et les caractres superflus pour obtenir une liste propre de labels uniques.

La fonction invert_multi_hot permet de convertir un vecteur multi-hot encod en une liste de labels lisibles, facilitant ainsi l'interprtation des rsultats du modle.

6- Préparation des jeux de données pour TensorFlow¶

Nous devons prparer les textes en assurant qu'ils ont une longueur unifie pour pouvoir les passer dans le modle. Pour cela, nous allons:Dfinir la longueur maximale de squence et le batch size, puis la fonction unify_text_length spare le texte en mots, calcule sa longueur et la quantit de padding ncessaire

Notre max_seqlen fix 400, alors tous les textes plus longs seront tronqus pour ne garder que les 400 premiers mots, et les textes plus courts seront remplis avec des tokens spciaux (comme ) pour atteindre cette longueur maximale.

Uniformiser la longueur des squences de texte pour qu'elles aient toutes la mme longueur, facilitant ainsi leur traitement par le modle de machine learning. En effet, pour qu'un batch de donnes puisse tre trait efficacement par un modle, chaque exemple dans le batch doit avoir la mme forme.

max_seqlen = 400
batch_size = 32
padding_token = "<pad>"
auto = tf.data.AUTOTUNE

def unify_text_length(text, label):
    word_splits = tf.strings.split(text, sep=" ")
    sequence_length = tf.shape(word_splits)[0]

    padding_amount = max_seqlen - sequence_length

    if padding_amount > 0:
        unified_text = tf.pad([text], [[0, padding_amount]], constant_values=padding_token)
        unified_text = tf.strings.reduce_join(unified_text, separator="")
    else:
        unified_text = tf.strings.reduce_join(word_splits[:max_seqlen], separator=" ")

    return tf.expand_dims(unified_text, -1), label

def make_dataset(dataframe, is_train=True):
    labels = tf.ragged.constant(dataframe["CPC_tronc"].values)

    dataset = tf.data.Dataset.from_tensor_slices(

    dataset = dataset.shuffle(batch_size * 10) if is_train else dataset
    dataset = dataset.map(unify_text_length, num_parallel_calls=auto).cache()
    return dataset.batch(batch_size)

8- Définition et compilation du modèle¶

Le modle de deep learning utilis dans votre programme est un rseau de neurones multicouche (MLP pour Multi-Layer Perceptron). Lorsque des donnes sont introduites dans le modle, elles passent travers chaque couche. import tensorflow.keras.backend as K

def recall_m(y_true, y_pred):
    y_true = tf.cast(y_true, tf.float32)
    y_pred = tf.cast(y_pred, tf.float32)
    true_positives = K.sum(K.round(K.clip(y_true * y_pred, 0, 1)))
    possible_positives = K.sum(K.round(K.clip(y_true, 0, 1)))
    recall = true_positives / (possible_positives + K.epsilon())
    return recall

def precision_m(y_true, y_pred):
    y_true = tf.cast(y_true, tf.float32)
    y_pred = tf.cast(y_pred, tf.float32)
    true_positives = K.sum(K.round(K.clip(y_true * y_pred, 0, 1)))
    predicted_positives = K.sum(K.round(K.clip(y_pred, 0, 1)))
    precision = true_positives / (predicted_positives + K.epsilon())
    return precision

def f1_m(y_true, y_pred):
    y_true = tf.cast(y_true, tf.float32)
    y_pred = tf.cast(y_pred, tf.float32)
    precision = precision_m(y_true, y_pred)
    recall = recall_m(y_true, y_pred)
    return 2*((precision*recall)/(precision+recall+K.epsilon()))
